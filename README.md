# case-framework-evidence

Evidence pipeline for the paper **"The CASE Framework: A Multi-Disciplinary
Control Architecture for Governing Agentic AI at Enterprise Scale"**. It
builds the datasets, LaTeX tables, and figure behind the paper's three
empirical studies from public sources.

| Study | Paper location | Emitted artifacts |
|---|---|---|
| 1. Failure taxonomy | Sec. 5.1, Table `tab:failures` | `results/study1_table.tex`, `data/study1_incidents.csv` |
| 2. Tooling capability matrix | Sec. 5.2, Table `tab:tooling` | `results/study2_table.tex`, `data/study2_tool_coding.csv` |
| 3. Enterprise maturity distribution | Sec. 5.3, Figure 1 | `results/figure1.pdf`, `data/study3_deployments.csv` |

Every headline statistic quoted in the paper is emitted to
`results/stats_summary.json`; every evidence item is logged in
`data/citation_ledger.csv` with its URL.

## Repository policy

**This repository ships the Python pipeline code only.** Generated data,
model-response caches, and the paper's data artifacts are *not* tracked here
(see `.gitignore`). They travel with the paper (in the paper's own folder:
`main.tex` alongside its `results/` and `data/`) and can be regenerated by
this pipeline. To run the pipeline you therefore need either the data bundle
(placed under `data/`) or an API key to regenerate it (below).

## Layout

```
src/
  collectors/   source collectors (AIID, MIT Risk Repo, arXiv, curated web) + screen + fetch + ledger
  coding/       Study 1 dual-coding: protocol.py (Appendix B rule base), llm_precoder, run_coding, adjudicate, kappa
  study2/       Study 2 tool-capability coder (code_tools.py)
  study3/       Study 3 composite index (index.py: geometric-mean, Eq. 9)
  emit/         LaTeX/JSON/figure emitters + build_study{1,2,3}_outputs orchestrators
  llm/          shared structured-output caller (SDK + stdlib HTTP fallback)
  run.py        Makefile entry points
config/         mechanism_classes.yaml, coding_protocol.yaml, sources.yaml, tools.yaml
tests/          pytest suite (run before claiming any task done)
notebooks/      adjudication_review.ipynb (the human-adjudication step of Study 1)
```

## Setup

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate     POSIX: source .venv/bin/activate
pip install -r requirements.txt        # Python 3.11+ (developed on 3.13)
```

Coding steps (Studies 1 and 2) call the Anthropic API for *uncached* items.
Put a key in a gitignored `.env`:

```
ANTHROPIC_API_KEY=sk-ant-...
```

## Reproduce

**Path A — from the shipped data bundle (no API key).** Copy the paper's
`data/` and `results/` into this repo's `data/` and `results/`, then
regenerate the tables and figure deterministically from cache:

```bash
make tables figures      # or: python -m src.run tables && python -m src.run figures
make test                # or: python -m pytest
```

**Path B — from scratch (needs an API key and re-fetches public sources).**

```bash
make refresh             # re-fetch AIID / MIT / arXiv / curated web (rate-limited, slow)
make study1              # dual-code candidates (API for uncached) -> pauses for adjudication
#   then review notebooks/adjudication_review.ipynb, fill data/study1_adjudications.csv, re-run make study1
make study2              # code the 22-tool registry from cached rendered docs -> table + stats
make study3              # score the deployment corpus -> figure1.pdf + stats
```

Coding is cache-first: raw model responses are written under
`data/raw/coding/`, `data/raw/tool_coding/`, so re-runs are free and
byte-identical. Approximate one-time API spend to regenerate all coding from
empty caches: Study 1 ~US$15-20, Study 2 ~US$2. Study 3 needs no coding API
(its corpus is scored in `data/study3_deployments.csv`).

## Commands

| make target | without make | purpose |
|---|---|---|
| `make candidates` | `python -m src.run candidates` | rebuild the Study 1 candidate pool from cache |
| `make refresh` | `python -m src.run refresh` | re-fetch all sources (rate-limited) |
| `make study1` | `python -m src.run study1` | code + adjudicate + emit Study 1 |
| `make study2` | `python -m src.run study2` | code + emit Study 2 |
| `make study3` | `python -m src.run study3` | score + emit Study 3 (figure) |
| `make tables` | `python -m src.run tables` | regenerate Study 1 + 2 tables from cache |
| `make figures` | `python -m src.run figures` | regenerate `figure1.pdf` from cache |
| `make all` | — | study1 + study2 + study3 + tables + figures |
| `make test` | `python -m pytest` | unit tests |

`make` is optional; the `python -m` commands are the source of truth on
Windows.

## Reviewer test

Every number the paper quotes is reachable in under a minute:

1. Find the statistic in `results/stats_summary.json` (keyed by study).
2. Trace it to its dataset row(s): `data/study1_incidents.csv` (incident,
   layer, rationale, `url`), `data/study2_tool_coding.csv` (tool, layer,
   verdict, evidence, `evidence_url`), or `data/study3_deployments.csv`
   (deployment, per-layer score, evidence, `url`).
3. Follow the URL to the public source. Source datasets and per-item
   provenance are also in `data/citation_ledger.csv`.

## Reproducibility contract

- Public sources only; robots.txt and terms respected; fetches rate-limited
  per `config/sources.yaml`.
- No fabrication: emitters fail loudly on empty or undersized inputs (Study 1
  refuses N < 50, Study 2 N < 15 tools, Study 3 N < 30 deployments).
- Determinism: the model API does not accept sampling parameters, so raw
  responses are cached and shipped with the data bundle; tables and the figure
  regenerate byte-identically from cache.
